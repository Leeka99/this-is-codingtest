# 이진 탐색

#### 순차 탐색
- 가장 기본 탐색 방법이다.
- 리스트 안에 있는 특정한 데이터를 찾기 위해 앞에서부터 데이터를 하나씩 차례대로 확인하는 방법이다.
- 보통 정렬되지 않은 리스트에서 데이터를 찾아야 할 때 사용한다.
- 리스트 내에 데이터가 아무리 많아도 시간만 충분하다면 항상 원하는 데이터를 찾을 수 있다는 장점이 있다.
- 시간 복잡도는 O(N) 이다.

#### 이진 탐색
- 탐색 범위를 반으로 좁혀가며 빠르게 탐색하는 알고리즘이다.
- 데이터가 정렬되어 있어야만 사용할 수 있는 알고리즘이다.
- 탐색하고자 하는 범위의 시작점, 끝점 그리고 중간점을 이용해 찾으려는 데이터와 중간점 위치에 있는 데이터를 반복적으로 비교해서 원하는 데이터를 찾는다.
- 처리해야 할 데이터의 개수나 값이 1000만 단위 이상으로 넘어가면 이진 탐색과 같이 O(logN)의 속도를 내야 하는 알고리즘을 떠올려야 문제를 풀 수 있는 경우가 많다.
- 시간 복잡도는 O(logN) 이다.

2. 반복문 사용

#### 데이터베이스에서 탐색과 이진 탐색 비교
- 데이터베이스는 내부적으로 대용량 데이터 처리에 적합한 트리 자료구조를 이용하여 항상 데이터가 정렬되어 있다.
- 따라서 이진 탐색과 유사한 방법을 이용해 탐색을 항상 빠르게 수행하도록 설계되어 있어서 데이터가 많아도 탐색하는 속도가 빠르다.

#### 트리 자료구조란?
- 데이터베이스 시스템이나 파일 시스템 같은 곳에서 많은 양의 데이터를 관리하기 위한 목적으로 사용한다.
- 노드와 노드의 연결로 표현한다.
- 노드는 정보의 단위로서 어떠한 정보를 가지고 있는 개체로 이해할 수 있다.
- 트리 자료구조는 그래프 자료구조의 일종이다.

#### 이진 탐색 트리
- 트리 자료구조 중에서 가장 간단한 형태가 이진 탐색 트리이다.
- 모든 트리가 다 이진 탐색 트리는 아니며, 이진 탐색 트리는 다음과 같은 특징을 가진다.
    - 부모 노드보다 왼쪽 자식 노드가 작다.
    - 부모 노드보다 오른쪽 자식 노드가 크다.
    - 간단하게 표현) 왼쪽 < 부모 < 오른쪽
