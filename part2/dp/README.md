# 다이나믹 프로그래밍(DP)

#### 컴퓨터를 활용해도 해결하기 어려운 문제?
- 최적의 해를 구하기에 시간이 매우 많이 필요한 문제
- 메모리 공간이 매우 많이 필요한 문제

#### 다이나믹 프로그래밍이란?
- 연산 속도와 메모리 공간을 최대한으로 활용할 수 있는 효율적인 알고리즘
- 한 번 계산한 문제는 다시 계산하지 않도록 한다.
- 동적 계획법이라고 표현하기도 한다.
- 다이나믹 프로그래밍으로 해결할 수 있는 대표적인 예시로 피보나치 수열이 있다.

#### 점화식?
- 인접한 항들 사이의 관계식을 의미한다.
- 점화식을 사용해 수열의 항이 이어지는 형태를 간결하게 표현한다.
수학적 점화식을 프로그래밍으로 표현하려면 재귀 함수를 사용하면 간단하다.

#### 다이나믹 프로그래밍을 사용할 수 있는 문제
1. 큰 문제를 작은 문제로 나눌 수 있다.
2. 작은 문제에서 구한 정답은 그것을 포함하는 큰 문제에서도 동일하다.

#### 
- 다이나믹 프로그래밍을 구현하는 방법 중 한 종류
- 
- 하는 것이다.
- 캐싱이라고도 한다.

#### 탑다운 방식
- 큰 문제를 해결하기 위해 작은 문제를 호출
- 재귀 함수를 이용하여 다이나믹 프로그래밍 소스코드를 작성하는 방법이다.
- 메모이제이션
    - 한 번 구한 결과를 메모리 공간에 메모해두고 같은 식을 다시 호출하면 메모한 결과를 그대로 가져오는 기법이다.
    - 한 번 구한 정보는 리스트에 저장한다.
    - 메모이제이션은 탐다운 방식에 국한되어 사용되는 표현이다.
    - 캐싱이라고도 한다.

#### 바텀업 방식
- 작은 문제부터 차근차근 답을 도출하는 방식
- 단순 반복문을 이용하여 소스코드를 작성하는 방식
- 다이나믹 프로그래밍의 전형적인 형태는 바텀업 방식이다.
- DP 테이블
