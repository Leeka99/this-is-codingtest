#### key Point
단순하게 푸는 방법이 아닌 효율적으로 푸는 방법을 생각하는 것이 중요했다.<br>
현재 풀이는 while문을 중첩시켜 단순하게 풀이했다.<br>
하지만 N이 100억 이상의 큰 수가 되는 경우에도 빠르게 동작하려면, N이 K의 배수가 되도록 효율적으로 한 번에 빼는 방식으로 코드를 작성하는 것이 좋다.
```
// 개선 코드
// ex: n : 23, k : 3

int target = (n / k) * k; // n -> 21라면 3으로 나눌 수 있음
result += (n - target); // 23 - 21 = 1 즉, 2번 빼는 것을 한번에 계산함

result += (n - 1) // k보다 작을 때 -1연산을 한번에 수행함

```
